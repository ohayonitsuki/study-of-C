* Введение в C (Introducing C)
** История происхождения
   + C был создан Деннисом Ритчи в начале 1970-х для разработки ОС UNIX. Язык задумывался как "высокоуровневый ассемблер", обеспечивающий эффективность при работе с железом.

** Важность стандартов
   + Кинг делает упор на использование стандартного C (ANSI C/C89 или C99), чтобы код был переносимым между разными компьютерами.

** Почему C до сих пор популярен
   + Эффективность 
     : C-программы быстрые и компактные. : 
   + Низкоуровневый доступ
     : Возможность манипулировать битами и адресами памяти. :
   + Портативность
     : Код, написанный согласно стандарту, легко переносится. :

** Слабая сторона
   + C не "защищает" программиста от ошибок (например, переполнение буфера), требуя высокой ответственности.

* Основы C (C Fundamentals)
** Структура простой программы:
   + Программа состоит из директив препроцессора (#include), функций (главная — main) и операторов (statements).

** Директивы препроцессора:
   + Строка #include <stdio.h> подключает библиотеку ввода-вывода до начала компиляции. Это необходимо для использования функций вроде printf.

** Функция main:
   + Каждая программа на C начинается с функции int main(void). Она возвращает целое число (0 — успех).

** Переменные и типы данных:
   + Переменные должны быть объявлены до использования. Основные типы: int (целое), float (с плавающей точкой).

** Форматированный вывод (printf):
   + Используется для вывода текста и значений переменных. Например, %d — для целых чисел, %f — для чисел с плавающей точкой.

** Форматированный ввод (scanf):
   + Используется для чтения данных от пользователя. Важно: перед именем переменной ставится знак амперсанда (&), кроме случаев чтения строк.

** Именованные константы (#define):
   + Вместо "магических чисел" в коде рекомендуется использовать #define NAME value, чтобы код был читаемым и его легко было менять.

** Комментарии:
   + Используются /* комментарий */ (классический стиль) или // комментарий (C99).

** Правила создания имен (идентификаторов):
   + Имена могут содержать буквы, цифры и подчеркивания, но не должны начинаться с цифры. C чувствителен к регистру (box и Box — разные имена).

* Функция printf
  + Компиляторы Си не обязаны проверять, совпадает ли число спецификаций преобразования в строке формата с числом выходных элементов. 
** Спецификация преобразования
   + В общем случаи спецификация преобразования может иметь вид %m.pX или %-m.pX, где m и p - целые константы, а X - буква  
     - d - отображает целое число в десятичном формате (основание 10).p указывает минимальное кол-во цифр для отображения (дополнительные нули добавятся к началу числа если это необходимо). Если p опущено, предполагается что оно имеет значение 1 (%d == %.1d)
     - e - отображает число с плавающей точкой в экспоненциальном формате (научная нотация).p показывает, сколько цифр должно появится после десятичной точки (по умолчанию 6). Если p равно 0, десятичная точка не отображется.
     - f - отображает число с плавающей точкой в "фиксированном десятичном формате" без покащателя степени.p имеет то же значение, что и для спецификатора e.
     - g - отображает число с плавающей точкой либо в экспоненциальном формате, либо в фиксированном десятичном формате в зависимости от размера числа.
        :: g не будет показывать конечные нули и если значение не имеет цифр после десятичной точки не отобразит десятичную точку. 
        :: Особенно полезен g для отображения чисел которые нельзя предсказать, когда программа пишется, или которые имеют тенденцию широко варьироваться по размеру.
